# Handoff — ServiceNow Now SDK: offline, no‑auth, update‑set delivery (complete knowledge base)

This document is a **self‑contained, project‑agnostic** knowledge base for building ServiceNow
scoped applications locally with the **Now SDK + Fluent**, and delivering them as **importable
Update Set XML** artifacts **without authenticating to any instance**.

It contains no client, project, vendor or scope specifics. Everywhere a project would put a real
value, this doc uses placeholders:

- `<scope>` → e.g. `x_<vendor>_<app>` (vendor prefix + app code), all lowercase.
- `<scopeId>` → a stable 32‑char hex sys_id for the scope.
- `<AppName>` → human application name.
- `<table>` → `<scope>_<entity>`.

A fresh AI session can read this file alone and reproduce the entire capability set.

---

## 0. The one idea

`now-sdk build` compiles Fluent metadata into ServiceNow XML records under `dist/app`. Those are
**not** a single importable update set. A small custom converter wraps each generated
`<record_update>` into a `<sys_update_xml>` payload inside one `<sys_remote_update_set>`. That XML
is imported on any instance via **Retrieved Update Sets → Import Update Set from XML → Preview →
Commit** — **no `now-sdk auth/install/deploy`, no credentials, no live connection**.

```
Fluent (.now.ts)  ──now-sdk build──▶  dist/app/*.xml  ──converter──▶  sys_remote_update_set.xml
                                                                          │
                                                  Retrieved Update Sets ◀─┘  (Preview → Commit)
```

The SDK ZIP from `now-sdk pack` is a **source/package archive**, NOT an update set. Never import the
ZIP through Retrieved Update Sets.

---

## 1. Non‑negotiable constraints (no‑auth mode)

Do **not** run, and do not ask the user for credentials:

```
now-sdk auth          # connect/login to an instance
now-sdk install       # push app to instance
now-sdk deploy        # deploy to instance
```

Everything is local build + offline XML packaging. The only way artifacts reach an instance is the
human importing the generated XML. This constraint holds unless the user explicitly lifts it.

## 6. The `explain` documentation system (mandatory before changing Fluent)

The SDK ships its own docs. **Always consult them before writing/altering any Fluent metadata** —
APIs evolve and `explain` is the source of truth.

```powershell
npx @servicenow/sdk explain --list --format=raw                 # list every topic + tags
npx @servicenow/sdk explain <topic> --list --peek --format=raw  # search topics, show descriptions
npx @servicenow/sdk explain <topic> --peek --format=raw         # preview a topic (do this first!)
npx @servicenow/sdk explain <topic> --format=raw                # read the full topic
```

Rules:
- **Never** open a full topic without `--peek` first (saves context, avoids wrong topic).
- The first time per session, skim everything under `explain quickstart --list`.
- Errors: `No documentation found for "<topic>"` = wrong name, use `--list`; `No match for "<topic>"`
  = try a different search term.

Useful search terms: metadata types (`Table`, `Acl`, `Role`, `ScriptInclude`, `BusinessRule`,
`ClientScript`, `UiAction`, `UiPolicy`, `Form`, `List`), surfaces (`workspace`, `serviceportal`,
`uipage`, `dashboard`, `ui-builder`), conventions (`naming`, `structure`, `scoping`, `file-layout`),
skills (`build`, `pack`, `transform`).

## 7. Fluent capability matrix (what builds to offline update‑set XML)

All of the following are declarable in Fluent and compile to `dist/app` XML with **no instance auth**.

| Capability | Fluent API | Record table | Notes |
|---|---|---|---|
| Tables & columns | `Table()` | `sys_db_object`, `sys_dictionary` | column types below |
| Choice lists | choice columns | `sys_choice` / `sys_choice_set` | value + label |
| Roles | `Role()` | `sys_user_role` | `containsRoles` for hierarchy |
| ACLs | `Acl()` | `sys_security_acl` (+ `_role`) | `type:'record'` and `type:'ux_route'` |
| App menu & modules | `ApplicationMenu()`, modules | `sys_app_application`, `sys_app_module` | backend nav |
| Workspace (Now Experience) | `Workspace()` | `sys_ux_*` (page registry, app config, route…) | landing path, tables |
| Workspace list config | `UxListMenuConfig()` | `sys_ux_list_menu_config`, `sys_ux_list*` | categories + lists + encoded queries |
| Visibility/applicability | `Applicability()` | `sys_ux_applicability` | role-based list/detail visibility |
| Dashboards | `Dashboard()` | `par_dashboard*` | widgets, single-score, charts, headings |
| Script Include | `ScriptInclude()` | `sys_script_include` | server JS via `Now.include` |
| Business Rule | `BusinessRule()` | `sys_script` | server logic |
| Client Script | `ClientScript()` | `sys_script_client` | classic UI |
| UI Action / UI Policy | `UiAction()`, `UiPolicy()` | `sys_ui_action`, `sys_ui_policy` | buttons, field rules |
| Form / List layout | `Form()`, `List()` | `sys_ui_form/section/element`, `sys_ui_list` | classic UI layout |
| Classic UI Page | `UiPage()` | `sys_ui_page` | Jelly/HTML or React (`@servicenow/react-components`) |
| Service Portal | `ServicePortal()` | `sp_portal` | urlSuffix → `/<suffix>` |
| Portal page | `SPPage()` | `sp_page` | containers→rows→columns→instances |
| Portal widget | `SPWidget()` | `sp_widget` | server/client/html/css via `Now.include` |
| Portal theme / menu / provider | `SPTheme()`, `SPMenu()`, `SPAngularProvider()` | `sp_theme`, `sp_instance_menu`, `sp_angular_provider` | branding/nav/directives |
| Flows | `Flow()` | `sys_hub_flow` | (consult explain) |
| ATF tests | `AtfTest()` | `sys_atf_test` | (consult explain) |

**NOT supported in Fluent (require the UI Builder editor on a live instance → break no‑auth):**
custom **UX Builder pages/components/macroponents** (`sys_ux_page`, `sys_ux_component`,
`sys_ux_macroponent`). If a project needs a fully custom single‑page experience offline, build it as
**Service Portal** (SPPage + SPWidget) or a classic **UiPage** — both are 100% offline‑packageable.

### Column types (Fluent `Table()`)
`StringColumn`, `MultiLineTextColumn`, `ChoiceColumn` (value/label sets), `BooleanColumn`,
`IntegerColumn`, `DateColumn` / date‑time, `ReferenceColumn` (→ another table, with `cascadeRule`),
`ListColumn` / Glide List (multi‑value reference, stored as comma‑separated sys_ids). Define
`index`/composite indexes for lookup/filter fields.

### Cross‑cutting helpers
- `Now.ID['stable_key']` → stable `$id` so re-builds produce the same sys_id (deterministic delivery).
- `Now.include('relative/path.ext')` → inline an external file (server JS, client JS, HTML, CSS/SCSS)
  into a record field.
- `Now.ref(...)` → reference another defined record.
- `index.now.ts` must `export * from './<each>.now'` so every metadata file is included in the build.

## 8. Service Portal specifics (hard‑won; read before building SP)

These cost real debugging time — internalize them:

1. **`SPPage` takes no top‑level `$id`.** It is keyed by `pageId`. The nested `containers`, `rows`,
   `columns` and `instances` **do** use `$id: Now.ID[...]`. A top‑level `$id` on SPPage fails the build.
2. **Widget client script must be a bare named function**: `function controller() { var c = this; … }`.
   The form `api.controller = function(){…}` is **rejected** by the SDK build. Dependencies are
   injected as parameters: `function controller(spModal) { … }`.
3. **Browser globals are blocked** in client scripts (`no-restricted-globals`): no `window`,
   `document`, `alert`, `confirm`. Use injected AngularJS services instead (e.g. `spModal.confirm()`,
   `spModal.alert()`).
4. **`customCss` is served AS‑IS and is NOT Sass‑compiled.** Put **plain CSS** in the `.scss`/css
   file: no nesting, no `$variables`, no `@each`, no `#{}` interpolation — any Sass syntax makes the
   browser discard the **entire** stylesheet (symptom: widget renders as unstyled plain text). For
   theming use **CSS custom properties** (`--var`) set on modifier classes; they are valid plain CSS.
5. **Bootstrap 3** classes (`panel`, `btn btn-*`, `nav nav-tabs`, grid) are available globally in
   Service Portal — fine for structure, but don't rely on them if you want a fully custom look.
6. Widget server script runs server‑side in the app scope: it can call same‑scope Script Includes
   directly (no GlideAjax). `data.*` is sent to the client; client `c.data.*` + `c.server.update()`
   round‑trips to the server script where the posted `data` arrives as `input`.
7. SP widget record fields seen in build output: `template` (html), `client_script`, `script`
   (server), `css`, plus `option_schema`, `demo_data`, etc.

Widget instance binding inside an `SPPage` container references the widget by its exported
`SPWidget(...)` const (the SDK resolves it to the widget sys_id).

## 9. Scoped GlideRecord, security & common platform patterns

- **Scoped GlideRecord enforces ACLs by default**, so a Service Portal widget that creates/updates
  via `new GlideRecord('<table>')` respects the record ACLs you defined in Fluent — no bypass.
- **Logical deletion** pattern: every business table carries `active` (Boolean, default true).
  Consumers and automations filter `active=true`; "deleting" sets `active=false`. Document this so
  inactive records are never shown in consultation nor used by automatic procedures.
- **Auto‑number** field: add a `number` String column and configure platform Number Maintenance
  (`sys_number`) per table for system‑generated identifiers.
- **Retention / data‑minimization** pattern: a `Date` column (e.g. `retention_date`) on entities with
  personal data; a configurable retention period (system property); a report/scheduled job listing
  expiring/expired records; UI Actions to revalidate (extend) or delete (single/bulk).
- **Scope protection / selective cross‑scope access**: set the application to restricted runtime
  access and grant cross‑scope access (e.g. a public/`All scopes` Script Include) only to explicitly
  authorized consumer scopes — not public‑to‑all. Configure `ScriptInclude` `accessibleFrom`
  accordingly and add cross‑scope privileges only for the named consumers.
- **Downloadable HTML export** pattern (offline‑friendly): a Service Portal widget whose server
  script renders the data and a client action that builds a self‑contained HTML string and triggers a
  browser download (Blob) — yields a static snapshot usable offline / for business continuity.

ACL pattern that builds cleanly:
- `type:'record'` ACLs per table per operation (`read`/`create`/`write`/`delete`), `read` to
  reader+editor, write‑ops to editor; `adminOverrides:true`.
- `type:'ux_route'` ACL `name:'<workspace-path>.*'`, `operation:'read'` for workspace route access.

## 10. The offline converter (`tools/sdk-dist-to-update-set.js`) — full generic source

Reads every `<record_update>` XML under `dist/app`, wraps each into a `<sys_update_xml>` inside one
`<sys_remote_update_set>`. **Table‑agnostic** (works for any Fluent output). Includes the `sys_app`
record so the target instance does **not** need the scoped app pre‑created. Supports a **delta**
whitelist via `--include`.

```js
#!/usr/bin/env node
const crypto = require('crypto')
const fs = require('fs')
const path = require('path')

// Friendly <type> labels in the sys_update_xml (cosmetic; unknown tables fall back to table name).
const TABLE_TYPES = {
  sys_app: 'Custom Application', sys_store_app: 'Store Application', sys_user_role: 'Role',
  sys_security_acl: 'Access Control', sys_script: 'Business Rule', sys_script_client: 'Client Script',
  sys_script_include: 'Script Include', sys_ui_action: 'UI Action', sys_ui_policy: 'UI Policy',
  sys_app_module: 'Module', sys_app_application: 'Application Menu', sys_db_object: 'Table',
  sys_dictionary: 'Dictionary Entry', sys_ux_page_registry: 'Workspace',
  sys_ux_list_menu_config: 'Workspace List Menu', sys_ux_applicability: 'Applicability',
  par_dashboard: 'Dashboard',
}

// Records intentionally NOT shipped offline. sys_user_role_contains causes non-blocking duplicate-key
// warnings on re-commit (uniqueness is on the role pair, not the row sys_id). Keep role hierarchy in
// Fluent source; validate containment manually post-install.
const EXCLUDED_UPDATE_TABLES = new Set(['sys_user_role_contains'])

function usage() {
  console.error('Usage: node tools/sdk-dist-to-update-set.js <dist/app> <output.xml> [name] [--include=spec,...]')
  console.error('  --include  Delta mode: comma list of exact table names or prefixes ending with "*"')
  console.error('             (e.g. --include=sp_*,sys_script_include). Only matching records emitted;')
  console.error('             sys_app is used for header metadata but emitted only if it matches.')
  process.exit(2)
}

const rawArgs = process.argv.slice(2)
const flagArgs = rawArgs.filter((a) => a.startsWith('--'))
const positional = rawArgs.filter((a) => !a.startsWith('--'))
const [distAppDirArg, outputFileArg, updateSetNameArg] = positional
if (!distAppDirArg || !outputFileArg) usage()

const includeFlag = flagArgs.find((a) => a.startsWith('--include='))
const includeSpec = includeFlag
  ? includeFlag.slice('--include='.length).split(',').map((s) => s.trim()).filter(Boolean)
  : []
const deltaMode = includeSpec.length > 0
const isIncludedTable = (t) =>
  !deltaMode || includeSpec.some((s) => (s.endsWith('*') ? t.startsWith(s.slice(0, -1)) : t === s))

const distAppDir = path.resolve(distAppDirArg)
const outputFile = path.resolve(outputFileArg)
if (!fs.existsSync(distAppDir) || !fs.statSync(distAppDir).isDirectory())
  throw new Error(`Input directory not found: ${distAppDir}`)

const guid = () => crypto.randomBytes(16).toString('hex')
const serviceNowDate = (d = new Date()) => {
  const p = (v) => String(v).padStart(2, '0')
  return `${d.getFullYear()}-${p(d.getMonth() + 1)}-${p(d.getDate())} ${p(d.getHours())}:${p(d.getMinutes())}:${p(d.getSeconds())}`
}
const escapeXml = (v) => String(v ?? '').replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;').replace(/"/g, '&quot;').replace(/'/g, '&apos;')

function listXmlFiles(dir) {
  const files = []
  for (const e of fs.readdirSync(dir, { withFileTypes: true })) {
    const fp = path.join(dir, e.name)
    if (e.isDirectory() && e.name === 'dictionary') continue   // skip SDK dictionary folder
    if (e.isDirectory()) files.push(...listXmlFiles(fp))
    else if (e.isFile() && e.name.toLowerCase().endsWith('.xml')) files.push(fp)
  }
  // build the scope record first
  return files.sort((a, b) => {
    const as = a.includes(`${path.sep}scope${path.sep}`) ? 0 : 1
    const bs = b.includes(`${path.sep}scope${path.sep}`) ? 0 : 1
    return as - bs || a.localeCompare(b)
  })
}

const tagValue = (xml, t) => {
  const m = xml.match(new RegExp(`<${t}(?:\\s[^>]*)?>([\\s\\S]*?)<\\/${t}>`))
  return m ? m[1].replace(/^<!\[CDATA\[([\s\S]*)\]\]>$/u, '$1').trim() : ''
}
const tagDisplay = (xml, t) => {
  const m = xml.match(new RegExp(`<${t}\\s+[^>]*display_value="([^"]*)"[^>]*>`))
  return m ? m[1] : ''
}

function parseRecord(fp) {
  const xml = fs.readFileSync(fp, 'utf8').replace(/^﻿/, '')
  const tm = xml.match(/<record_update\b[^>]*\btable="([^"]+)"/) || xml.match(/<record_update\b[^>]*>\s*<([a-z0-9_]+)\b/i)
  if (!tm) throw new Error(`Not a record_update XML file: ${fp}`)
  const table = tm[1]
  const sysId = tagValue(xml, 'sys_id') || guid()
  const updateName = tagValue(xml, 'sys_update_name') || `${table}_${sysId}`
  const targetName = tagValue(xml, 'sys_name') || tagValue(xml, 'title') || tagValue(xml, 'name') || tagDisplay(xml, 'sys_scope') || updateName
  return { xml, table, sysId, updateName, targetName, type: TABLE_TYPES[table] || table, replaceOnUpgrade: tagValue(xml, 'sys_replace_on_upgrade') || 'false' }
}

const parsed = listXmlFiles(distAppDir).map(parseRecord)
const excluded = parsed.filter((r) => EXCLUDED_UPDATE_TABLES.has(r.table))
const records = parsed.filter((r) => !EXCLUDED_UPDATE_TABLES.has(r.table))
if (records.length === 0) throw new Error(`No XML files found under ${distAppDir}`)

const appRecord = records.find((r) => r.table === 'sys_app') || records.find((r) => r.table === 'sys_store_app')
if (!appRecord) throw new Error('No sys_app/sys_store_app XML found. Not a complete application package.')

const emitted = records.filter((r) => isIncludedTable(r.table))
if (deltaMode && emitted.length === 0) throw new Error(`No records matched --include=${includeSpec.join(',')}`)

const appXml = appRecord.xml
const appId = tagValue(appXml, 'sys_id') || appRecord.sysId
const appScope = tagValue(appXml, 'scope') || tagValue(appXml, 'sys_code') || 'global'
const appName = tagValue(appXml, 'name') || updateSetNameArg || appScope
const appVersion = tagValue(appXml, 'version') || '1.0.0'
const updateSetName = updateSetNameArg || `${appName} ${appVersion}`
const now = serviceNowDate()
const remoteUpdateSetId = guid()
const remoteSysId = guid()

const lines = ['<?xml version="1.0" encoding="UTF-8"?>', `<unload unload_date="${escapeXml(now)}">`,
  '<sys_remote_update_set action="INSERT_OR_UPDATE">',
  `<application display_value="${escapeXml(appName)}">${escapeXml(appId)}</application>`,
  `<application_name>${escapeXml(appName)}</application_name>`,
  `<application_scope>${escapeXml(appScope)}</application_scope>`,
  `<application_version>${escapeXml(appVersion)}</application_version>`,
  '<collisions/>', '<commit_date/>', '<deleted/>',
  `<description>Generated offline from SDK build output at ${escapeXml(now)}</description>`,
  '<inserted/>', `<name>${escapeXml(updateSetName)}</name>`, '<origin_sys_id/>', '<release_date/>',
  `<remote_sys_id>${escapeXml(remoteSysId)}</remote_sys_id>`, '<state>loaded</state>', '<summary/>',
  '<sys_created_by>admin</sys_created_by>', `<sys_created_on>${escapeXml(now)}</sys_created_on>`,
  `<sys_id>${escapeXml(remoteUpdateSetId)}</sys_id>`, '<sys_mod_count>0</sys_mod_count>',
  '<sys_updated_by>admin</sys_updated_by>', `<sys_updated_on>${escapeXml(now)}</sys_updated_on>`,
  '<update_set display_value=""/>', '<update_source display_value=""/>', '<updated/>',
  '</sys_remote_update_set>']

for (const r of emitted) {
  lines.push('<sys_update_xml action="INSERT_OR_UPDATE">', '<action>INSERT_OR_UPDATE</action>',
    `<application display_value="${escapeXml(appName)}">${escapeXml(appId)}</application>`,
    '<category>customer</category>', '<comments/>', `<name>${escapeXml(r.updateName)}</name>`,
    `<payload>${escapeXml(r.xml)}</payload>`,
    `<remote_update_set display_value="${escapeXml(updateSetName)}">${escapeXml(remoteUpdateSetId)}</remote_update_set>`,
    `<replace_on_upgrade>${escapeXml(r.replaceOnUpgrade)}</replace_on_upgrade>`,
    '<sys_created_by>admin</sys_created_by>', `<sys_created_on>${escapeXml(now)}</sys_created_on>`,
    `<sys_id>${guid()}</sys_id>`, '<sys_mod_count>0</sys_mod_count>', '<sys_updated_by>admin</sys_updated_by>',
    `<sys_updated_on>${escapeXml(now)}</sys_updated_on>`, '<table/>',
    `<target_name>${escapeXml(r.targetName)}</target_name>`, `<type>${escapeXml(r.type)}</type>`,
    '<update_domain>global</update_domain>', '<update_set display_value=""/>', '<view/>', '</sys_update_xml>')
}
lines.push('</unload>')

fs.mkdirSync(path.dirname(outputFile), { recursive: true })
fs.writeFileSync(outputFile, `${lines.join('\n')}\n`, 'utf8')

console.log(`Wrote ${outputFile}`)
if (deltaMode) {
  const byTable = emitted.reduce((a, r) => ((a[r.table] = (a[r.table] || 0) + 1), a), {})
  console.log(`Delta mode --include=${includeSpec.join(',')}`)
  console.log(`Included ${emitted.length} of ${records.length} record(s). By table: ` +
    Object.keys(byTable).sort().map((t) => `${t}=${byTable[t]}`).join(', '))
} else {
  console.log(`Included ${emitted.length} SDK XML record(s).`)
}
if (excluded.length) console.log(`Excluded ${excluded.length}: ${[...new Set(excluded.map((r) => r.table))].join(', ')}.`)
```

### Delta packaging
`--include=sp_*,sys_script_include` emits only matching tables (prefix `*` or exact). Use a delta when
the base app is **already committed** on the target and you only want to ship a subset (e.g. just the
portal + a changed Script Include). The header still binds to the existing app via the `sys_app`
metadata, but the `sys_app` record itself is not emitted. Roles/tables/ACLs the delta references must
already exist on the instance. A full re‑import (no `--include`) is cumulative and idempotent
(`INSERT_OR_UPDATE` by sys_id); data rows are never in the update set, so operational data is untouched.

## 11. Seed / demo data (separate from the app)

Seed data is a **ServiceNow XML data unload**, imported as **data**, not as an update set. Generate it
with `tools/generate-seed-data.js` writing `<unload>` with one block per record, **deterministic
sys_ids** (e.g. md5 of a natural key), and only **non‑personal** demo values (`example.invalid`
emails, fake numbers). Import via: open any list → right‑click a column header → **Import XML**.
**Never** upload seed XML through Retrieved Update Sets.

Skeleton of a record block inside `<unload unload_date="…">`:
```xml
<x_scope_table action="INSERT_OR_UPDATE">
  <sys_id>deterministic-32-hex</sys_id>
  <name>Demo value</name>
  <active>true</active>
  ...fields...
</x_scope_table>
```

## 12. Validators (TDD discipline — add/adjust BEFORE risky packaging changes)

Three plain Node scripts; they make the workflow trustworthy. Add assertions before implementing a
risky packaging behavior, then implement until green.

- **`validate-source.js`** — checks the Fluent source: `now.config.json` exists; expected scope/name;
  `scopeId` is 32‑hex; all expected tables/roles/choices present; workspace path + route ACL present;
  Script Include metadata present; **old scope absent**; (project‑specific) presence of key methods,
  asset files, and behavioral markers (e.g. a client‑side filter token) to prevent regressions.
- **`validate-update-set.js <xml>`** — checks the packaged XML: `<sys_remote_update_set>` present;
  app scope + `sys_app_<scopeId>` present; expected table names present; workspace/route records
  present; **old scope & old scopeId absent**; `sys_user_role_contains` absent; enough
  `<sys_update_xml>` (use `>=` thresholds, never brittle equality). **Match the ESCAPED form**: records
  live inside `<payload>` HTML‑escaped, so a table tag appears as `&lt;<table>` and each maps to a
  `<name><table>_<32hex></name>`. Assert on `&lt;sp_portal`, counts via `/<name>sp_page_[0-9a-f]{32}<\/name>/g`, etc.
- **`validate-seed-data.js <xml>`** — unload format present; exact record counts; sample values; unique
  sys_ids; role containment not altered.

## 15. Troubleshooting (symptom → cause → fix)

| Symptom | Cause | Fix |
|---|---|---|
| SP widget renders as unstyled plain text | Sass syntax in `customCss` (served as plain CSS, not compiled) | Write **plain CSS** only; theme via CSS custom properties (`--var`) |
| Build error "Client controller must contain a JavaScript function" | Used `api.controller = function(){}` | Use bare `function controller() { var c = this; … }` |
| Build error: `window`/`document` not allowed | Browser global in client script | Use injected services (`spModal`, etc.) |
| Build error: `$id` not allowed on SPPage | SPPage keyed by `pageId` | Remove top‑level `$id` (keep `$id` on containers/rows/columns/instances) |
| Duplicate‑key warnings on commit for role containment | `sys_user_role_contains` uniqueness on role pair | Exclude it from the update set (already in converter); validate containment manually post‑install |
| Validator can't find `<sp_portal>` / records | Records are HTML‑escaped inside `<payload>` | Match `&lt;sp_portal`, count via `<name><table>_<32hex></name>` |
| Need a custom single‑page UI but no instance auth | UX Builder pages aren't Fluent/offline | Use Service Portal (SPPage+SPWidget) or classic UiPage |
| SDK ZIP rejected as update set | ZIP is a source/package archive | Import the generated `sys_remote_update_set` XML instead |
| `explain` not found / errors | SDK < 4.6 or wrong topic | Upgrade SDK; use `--list` to find the right topic |
